MANGO.SYS Interactive Security Portfolio

A dependency-free professional portfolio paired with an optional security-themed canvas raycaster.

Problem

A conventional résumé site would communicate experience clearly but lose Ryan’s playful mango/cybersecurity identity. A game-only landing page would be memorable but make recruiters work too hard to find professional qualifications.

Constraints

  • Static hosting with no database or application backend
  • Vanilla HTML, CSS, and JavaScript
  • Professional information must remain available without JavaScript
  • Responsive behavior from small phones through desktop displays
  • No invented outcomes, employer-confidential data, or unsupported claims

Investigation and Implementation

The site uses semantic HTML as its baseline. JavaScript enhances hash navigation, focus movement, the optional boot sequence, and the game, while the résumé and project content remain ordinary linkable documents.

The playable mission uses a two-dimensional grid, DDA raycasting for walls, billboard sprites, a depth buffer for occlusion, hitscan combat, collision detection, and lightweight enemy pursuit. The interface adds a mission briefing, pause lifecycle, keyboard controls, status announcements, and reduced-motion support around that engine.

Tools

Semantic HTML, responsive CSS, vanilla JavaScript, Canvas 2D, DDA raycasting, Git, and static web hosting.

Outcome

The current result is a responsive cybersecurity portfolio whose core résumé, project, and contact paths do not depend on the decorative boot experience. Visitors who want the playful layer can launch a complete single-level browser mission from the same site.
